# 工作流全环节状态与安全机制规范

本文档系统性地梳理了当前应用在分集、资产、分镜、视频等各环节中针对“每集（Episode）”和“实体（Entity）”所采用的所有状态机（State Machine）字段及其含义，以及相关的安全控制逻辑。

---

## 1. 核心集数管理状态 (Episode State)
每次分集（Step 1）产生的新集数，会在底层平行数组 `activeProject.epState` 中进行独立的状态追踪。

### 1.1 资产提取状态 (`epState.s2[i]`)
此状态严格控制下游的分镜与视频是否允许生成。

| 状态值 | 含义 | 触发时机 |
| --- | --- | --- |
| `''` (空) 或 `pending` | **未提取** | 刚切分出的全新集数，或被安全清空原文的槽位。 |
| `running` | **提取中** | 点击“资产分析”，开始执行名称提取（2a）或详情提取（2b）时。 |
| `done` | **已完成** | 该集的所有角色、场景、物品资产都已经成功提取并合并，无任何报错。 |
| `error` | **提取失败** | 遇到了严重的API错误、超时、或返回了完全无法解析的乱码格式，导致流程彻底崩溃中断。 |
| `partial` | **部分完成** | 在并发提取多个资产时，有一部分资产成功，但有少部分资产失败（被记录在 `s2FailedTasks` 中）。支持点击“继续未完成”。 |

### 1.2 下游安全放行标记 (`epState.ready[i]`)
| 状态值 | 含义 | 触发时机 |
| --- | --- | --- |
| `false` | **未就绪 (拦截)** | 初始状态，或者 `s2[i]` 不为 `done` 时。如果用户强行执行分镜，系统会拦截。 |
| `true` | **已就绪 (放行)** | 只有当该集的 `s2` 严格达到 `done` 时才会自动切换为 `true`。 |

## 2. 全局资产实体内部状态 (Entity State)
这些状态存在于 `activeProject.assets.characters` / `scenes` / `items` 数组的每一个独立对象中。

| 字段 / 状态值 | 含义 | 触发时机与表现 |
| --- | --- | --- |
| `_pending: true` | **等待详情分析** | 2a名称提取完毕，只拿到了一个名字骨架，但2b还没跑或失败了。 |
| `_status: 'extracting'` | **提取资料中...** | 对于一个全新的卡片，刚刚开始跑2b提取详情。UI面板上显示黄色提示。 |
| `_status: 'updating'` | **资料更新中...** | 对于一个老卡片（已有详情），因为追加了新集数重新跑2b来补充新造型。UI面板上显示黄色提示。 |
| `_updates: [...]` | **增量更新记录** | 2b跑完后，如果对比发现了新增加的造型/形态/状态，记录在这里。UI面板绿字展示 `最近更新: 造型:战斗服`。 |

---

## 3. 分镜生成状态 (Storyboard State)
分镜是一个包含子流程的复杂步骤，分为基础骨架、智能归并（Script模式独有）、以及镜头裂变生成。

### 3.1 基础分镜生成 (`epState.sb[i]`)
| 状态值 | 含义 |
| --- | --- |
| `''` (空) | **未生成** |
| `running` | **生成中** |
| `done` | **已生成** |
| `error` | **生成失败** |

### 3.2 分镜头裂变生成 (`epState.sbShots[i]`)
| 状态值 | 含义 |
| --- | --- |
| `''` (空) | **未裂变** |
| `running` | **镜头创作中** (UI: 进度条转动) |
| `done` | **创作完成** (UI: 显示“重写全部镜头”等按钮) |
| `error` | **创作失败** |

---

## 4. 视频与提示词状态 (Video Prompt State)

### 4.1 视频提示词生成 (`epState.s4c[i]`)
视频提示词强依赖该集的分镜（`episodeStoryboard`）必须存在。

| 状态值 | 含义 |
| --- | --- |
| `''` (空) | **待生成** |
| `running` | **生成中** |
| `done` | **已生成** (此时可将数据推送到实际渲染队列) |
| `error` | **生成失败** |

---

## 5. 空槽位状态逻辑回顾 (Empty Slot Logic)
当用户在“分集页面”点击红色的垃圾桶时，不会破坏平行数组结构。
- **操作**：`title` 和 `text` 置为 `''`。
- **UI 呈现**：标题前出现红色的 `[空]` 标签。
- **数据保留**：该集的 `s2`，`sb`，以及其贡献的全局资产都不会被清理。
- **填补机制**：当上传了任何未识别的新剧本，将优先填入第一个带有 `[空]` 标记的槽位，之后再追加到最后面。新填补的槽位状态将立刻退回 `pending`，要求重新提取资产。
